The current benchmark design still has real limitations. Native adversarial coverage outside English remains sparse, so some language coverage will initially depend on translation and deterministic masking rather than organically collected attacks. Phase 1 of the benchmark is also primarily single-turn, which means gradual multi-turn prompt injection remains only partially captured until the session pipeline lands.

There are also interface confounds. Different providers expose different token accounting, hidden routing behavior, and policy surfaces. If these are not normalized carefully, the benchmark can accidentally measure provider wrappers instead of prompt-classifier behavior. Finally, adversarial prompt styles change quickly. A benchmark grammar is more resilient than a frozen CSV, but temporal drift remains part of the problem and should be acknowledged explicitly.
